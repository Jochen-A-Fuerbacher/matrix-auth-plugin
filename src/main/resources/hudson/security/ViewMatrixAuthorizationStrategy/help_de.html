Dieser Modus erweitert die "Matrix-basierte Sicherheit" um eine zusätzliche ACL-Rechtematrix, die für jede View individuell festgelegt werden kann (in der Konfigurationsmaske der jeweiligen View).

Dies erlaubt Ihnen eine Rechtevergabe wie z.B. "Joe darf auf View A, B und C zugreifen - er soll aber View D nicht sehen dürfen". Lesen Sie den Hilfetext bei "Matrix-basierte Sicherheit", um mehr über das Konzept der matrix-basierten Sicherheit im Allgemeinen zu erfahren.

ACL-Rechte sind additiv. Dies bedeutet, dass die untenstehend erteilten Rechte für alle Views als erteilt gelten.
